## ğŸŒ GitHub Pages Deployment

### Method 1: Direct GitHub Pages Setup

1. **Create a New Repository**
   - Go to GitHub and create a new repository
   - Name it `yourusername.github.io` (for user site) or any name (for project site)
   - Make sure it's public

2. **Upload Your Files**
   - Upload all files to your repository
   - Ensure `index.html` is in the root directory

3. **Enable GitHub Pages**
   - Go to your repository settings
   - Scroll down to "Pages" section
   - Under "Source", select "Deploy from a branch"
   - Choose "main" branch and "/ (root)" folder
   - Click "Save"

4. **Access Your Site**
   - Your site will be available at: `https://yourusername.github.io/` (user site) or `https://yourusername.github.io/repository-name/` (project site)
   - It may take a few minutes for the site to be live

### Method 2: Using Git Commands

1. **Initialize Git Repository**
   ```bash
   git init

   git commit -m "Initial commit: Add portfolio website"
   ```

2. **Connect to GitHub**
   ```bash
   git remote add origin https://github.com/yourusername/repository-name.git
   git branch -M main
   git push -u origin main
   ```

3. **Enable GitHub Pages** (follow steps 3-4 from Method 1)
